CA117 Checklist/Guide Thing

Here’s what I think will come up in the Programming II exam.

Question 1.

  • Basic string slicing
  • Slice parameters [start (inclusive) : end (exclusive) : step]

Question 2.

  • Mutable types e.g. dict, set, list...
  • Immutable types e.g. int, str, tuple, namedtuple...
  • Slicing a list to copy it e.g. newlist = oldlist[:]

Question 3.

  • Basic list comprehensions e.g. [line.strip() for line in sys.stdin]
  • Adding an if statement e.g. [n for n in numberlist if n%2==0] (only appends the even numbers)

Question 4.

  • Default parameters e.g. def do_stuff(thing1, thing2=0) (thing2 is a default parameter)
  • Jumbled up parameters e.g. def do_stuff(a, b) … do_stuff(b=10, a=20)

Question 5.

  • Regular expressions. Dear christ…
  • Regex methods e.g. findall, search, match...
  • Default character classes e.g. \d, \D, \s, \S, \w, \W, \b, \B...
  • Using (?:blah) to prevent python from returning tuples
  • Using * (0 or more times)
  • Using + (1 or more times)
  • Using ? (0 or 1 times)

Regex isn’t really that difficult. Regexr (an online regex tool) helps a lot.
Go watch some youtube videos. I think sentdex is the name of a guy who did a good regex tutorial.

Question 6.

  • UML Diagrams (Just those class diagrams from the lab exams)
  • Inheritance e.g. class Cat(Animal)...
  • Polymorphism e.g. cat.speak() and dog.speak() (Same function, different behaviour) Video explanation.
  • Class variables e.g. File.DEFAULT_PERMISSIONS
  • Instance/Member variables e.g. guy = Guy("Bob") … print(guy.name)
    • ^^^ Remember... Instance variables are defined using self.foo = bar
  • Static methods

Question 7.

  • Recursive quicksort function (without partition step)

Question 8.

  • Partition function (for quicksort)
  • Understand output of partition function

Question 9 and 10. (Long questions)

  • More class stuff

Other things to memorise

  • Reading files as text f = open("bernie_sanders.txt", "rt")
  • Writing files as text f = open("bernie_sanders.txt", "wt")
  • Reading files as binary f = open("bernie_sanders.txt", "rb")
  • Writing files as binary f = open("bernie_sanders.txt", "wb")
  • Reading standard input (sys.stdin)
  • Operator overloading (very important)

If you’re worried about this exam, just remember that there’s not that much you need to know.
If you’re properly stuck just post a message in the CA facebook group and someone’ll help you out.
